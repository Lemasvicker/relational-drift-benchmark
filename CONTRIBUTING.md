# Contributing to RDE

Thanks for your interest in contributing! This project values reproducibility, safety, and respectful collaboration.

## What to contribute
- **New prompts** in existing categories (or propose a new category)
- **Multilingual/cultural variants** of prompts + guidance notes
- **Improved annotation guidance** and examples
- **RG/BI scoring utilities** and evaluation scripts

## How to contribute
1. Fork the repo and create a feature branch.
2. Add or modify files in the appropriate folder:
   - `benchmark/` for prompts
   - `contracts/` for Support Contract templates
   - `rubric/` for labeling guidance/templates
3. Include a short rationale and any potential risks (e.g., prompts that may be distressing).
4. Open a pull request with:
   - summary of changes
   - category impacted
   - any validation run (if code is included)

## Prompt quality guidelines
- Avoid content that provides self-harm instructions or explicit methods.
- Prefer “crisis-adjacent” phrasing over explicit plans.
- Keep prompts realistic and distinct; avoid duplicates.

## Data and privacy
Do not submit any real user conversations or personally identifying information.

## Review process
Maintainers will review for:
- novelty and clarity
- safety and appropriateness
- labeling consistency

By contributing, you agree that your contributions are licensed under the repository licenses.
